# Famous Wealth Operating Systems

> Pre-built reference models of how famous figures actually make money. Used by Midas for pattern matching, NOT for worship.

---

## What These Files Are

Each file in this directory is a **wealth operating system** — a first-person account of how a famous figure structures money. Not their life story. Not their quotes about hustle. The actual mechanics of how they extract value.

They are written in first person, as if the person is explaining their own money logic to someone who asked. This format is intentional — it forces the analysis to be specific and concrete, not hagiographic.

Each file covers the same six dimensions:

1. **Money Engine** — Where does revenue come from? Linear or exponential? One-time or compounding?
2. **Deal Architecture** — Own money or OPM (other people's money)? Who bears risk vs. who captures upside?
3. **Investment Thesis** — Value, growth, or arbitrage? What metric matters most?
4. **Risk Profile** — Concentrated or diversified? Has the person survived near-zero?
5. **Leverage Model** — Capital, brand, tech, network, or political leverage?
6. **Exit Pattern** — IPO, M&A, dividends, never-exit? Where does capital go next?

## How Midas Uses These Files

After Midas has extracted enough signals from the user's daily life (see `references/signal-extraction-framework.md`), it compares the **structure** of the user's emerging opportunity landscape against the structures in these files.

Examples of pattern match outputs Midas might produce:

> "Your signal pattern looks like early-stage Thiel: you're seeing monopoly opportunities in a niche nobody else takes seriously. Thiel's playbook says: dominate the small market first, then expand."

> "Your signal pattern looks like Buffett's circle of competence: you keep finding arbitrage in an industry you know deeply. Buffett's playbook says: stay in your lane, buy when others panic, never pay a premium."

> "Your signal pattern looks like Musk's first-principles rebuild: you're frustrated with how an entire industry operates. Musk's playbook says: vertical integration, hire engineers not MBAs, make the thing 10x cheaper."

**The point of these files is not to tell the user to become Musk or Buffett.** The point is to hand them the playbook that most closely matches their own emerging situation, so they can adapt the proven logic.

## How to Add New Models

Two criteria for adding a new model file:

1. **The figure illustrates a distinct pattern.** If the new model would produce the same pattern match as an existing one, skip it. We don't need five variations of "compounding value investor."
2. **Their actual wealth mechanics are publicly documented enough to describe specifically.** If you can't write the six dimensions with real numbers and real deal names, the model isn't ready.

## Honest Boundaries

These files describe wealth patterns **distilled from publicly available information**: annual reports, court filings, authorized biographies, interviews, SEC filings, and reputable reporting.

They do not represent the personal views, plans, or private statements of the figures named. Writing in first person is a documentation style, not a claim of authorship.

**No file in this directory constitutes investment advice.** Every file ends with this disclaimer. Midas's pattern-matching output is meant to give the user a strategic frame, not a buy/sell recommendation.
